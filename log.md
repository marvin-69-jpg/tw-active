- 開新 mechanism page [[wiki/mechanisms/etf-transparency-frontrunning]]——把「揭露 vs 主動經理 IP 保護」trade-off 鋪好。核心觀察：台灣選 transparent 路線是法規路徑依賴而非 trade-off 決策，主動 fee 1.0-1.2% 結構與「IP 每日強制 leak」結構錯配。列 5 個可量化 testable hypotheses（H1 front-running effect / H2 BDR reversal 在台灣 / H3 規模 → IP-leak cost 反比 / H4 turnover → cumulative drag / H5 揭露 timestamp micro-timing）
- 實作 H1 prototype（`tools/frontrunning.py` + skill + docs）。從 raw/cmoney/shares/ 17 檔 TW-focused 主動 ETF 建 2058 個加碼事件（Δ% ≥ 5% 或新建倉，Δshares ≥ 100,000），對 250 檔個股從 FinMind 抓 Trading_Volume，算 vol(T)/median(vol[T-20:T-1])。pooled T median = 1.31（揭露日成交量比 baseline 高 31%），T+1 = 1.24，T+2 = 1.18 衰減清楚
- H1 v2：dump 5 檔被動 ETF（0050/0056/006208/00692/00891）持股到 `raw/cmoney/shares-passive/`，frontrunning.py 加 `--with-passive-control` flag。**反直覺結果**：passive pooled T median = 2.12 > active 1.31。揭露 → abnormal vol 是 generic ETF rebalance 效應（passive index reconstitution effect 早被學界記錄），主動 ETF 反而較弱。**H1 嚴格意義不成立**，敘事 angle 翻轉：主動 ETF 揭露 cost 比 passive 較低而非較高。可能解釋：(a) 高頻小幅多元調倉稀釋每次 front-running profit、(b) AP creation/redemption 雜音、(c) 反向因果（manager 是 momentum follower）。把 mechanism page H1 條目從「強支持」改成 nuanced finding
- 實作 H4 prototype（`tools/cumulative_drag.py` + skill + docs）。在 H1 v2 events 之上算「年化加總、AUM-normalize」cumulative metrics（excess_volume_shares 與 manager_drag），window-aligned 對齊到主動 ETF 11 個月窗口（被動 raw 800 日資料截掉前面）。FundClear `totalAv` 補被動 AUM（cmoney meta 沒收）。FinMind 配額用完，metric 留在「股」維度沒乘 close。**結果**：active drag/AUM = 775 vs passive 382 → ratio **2.03×**，**H4 weakly supported**——driver 是 events/yr 高 20.5×（5810 vs 284）。但 0056 一檔占 passive pooled drag ~70%，passive baseline 高度依賴單一樣本。比 v2 翻盤前期待的「壓倒性差距」溫和很多。mechanism page 加 H4 entry，下一步 refinement = same-stock matched pairs（H4'）
- 實作 H4'（`tools/matched_pairs.py` + skill + docs）。對 28 檔同時被 active 與 passive 加碼（每側 ≥2 events）的 overlap 股票做 same-stock paired comparison。**強結果**：86%（24/28）股票 passive median > active median，median diff = -0.99。**H1 v2 結論強化**——配對控制下 active 仍系統性較弱，stock mix 不是 confound。國巨 11.07、台灣大 5.12、中信金 6.31 等權值/金融股 passive 揭露日 vol 巨大（疑似 index reconstitution events）。對 mechanism page 做 synthesis：Haeberle 框架在台灣不直接成立，真正大的 transparency cost 在被動端，主動 ETF 結構性問題的研究方向應轉向 fee / 配息平準金 / 規模等其他路徑。送上意外 finding：「被動 ETF 揭露日 abnormal vol」自己值得開新 mechanism page
