# Threads 結構模板

6 種結構，每種配 archive 範例（可去 `reports/threads/archive.md` 看全文）。

---

## 單股 portrait

**用途**：講一個經理人對一檔股票的完整操作軌跡（建倉 / 加減碼 / 清倉）。

**結構**：
1. 一句 hook（標的 + 觀察點）
2. 時間線（每個關鍵時點的張數 + 成本 / 均價）
3. 對照（跟另一檔的對比 OR 跟過去的對比）
4. 紀律歸納（一兩句總結這位經理人的決策模式）

**長度**：300-450 字

**範例**：
- `18073223537558687` 鴻海三輪（2025-05 / 2025-09 / 2026-04）
- `18087483107521403` 聯發科九天 508→3,295 張
- `18049855544725160` 台光電 220 天加減碼

**模板**：
```
<標的代號 + 名字>。<觀察 hook>。

<時點1>: <張數/金額/均價/動作>。
<時點2>: <張數/金額/均價/動作>。
...

<總結這條軌跡的紀律或規律>。
```

## 反問 hook

**用途**：戳破流行說法 / 直覺，用數據反證。

**結構**：
1. 反問（流行說法 + 真的是這樣嗎？）
2. 「同一時間，她還押了另一筆」式的反例
3. 數據對照
4. 紀律總結 + 對讀者的 punchline

**長度**：250-400 字

**範例**：
- `18148167691427951` 「瑤姐就是市場主力嗎？」聯電打臉

**模板**：
```
有人說 <流行說法>，但真的是這樣嗎？

<反例事實 / 同一時間另一筆操作>。

<反例數據時間線>。

<把規律講出來>：贏的壓，不贏的不追，輸的砍。

<對跟單散戶的提醒，可帶 punchline>
```

⚠️ **00981A 重疊類用正向 framing**（共識中心，不抓 closet indexer）。

## 事件追蹤

**用途**：個股大漲 / 大跌時，看跨 ETF 怎麼動。

**結構**：
1. 事件 hook（標的 + 漲跌幅 + 時點）
2. 「上週之前已布局」清單
3. 「本週新進場 / 加碼」清單
4. 觀察（誰在順勢、誰在新建）

**長度**：250-400 字

**範例**：
- `17900970909262409` 聯發科 +7.5% 漲停，跨 6 檔 ETF 持倉
- `18523838680079173` 台積電大漲，00981A 4 月加碼 3,685 張

**模板**：
```
<標的> 從 <時點> 起漲（<幅度>），<最新時點><現價>。這段主動型 ETF 怎麼動？

上週之前就已布局（<截止日> 持有）：
・<ETF 名 代號>：<金額> / <權重>
・...

<本週新動作>。

<順勢 vs 主動的觀察>。
```

## 拆解

**用途**：拆解一個宏觀現象（規模 / 資金 / 結構），讓讀者看到組成。

**結構**：
1. 標題式 hook（「X 拆解 · 220 天」）
2. 分組數據（規模 / 申購 vs 漲幅 / 飛輪信號）
3. 「我本來以為 X，把 Y 拆開才看到 Z」
4. 開放問題

**長度**：350-500 字

**範例**：
- `18014612798833232` 00981A 自肥效應 220 天
- `18211121173325589` 主動 ETF 4/21 資金流（前 8 檔吃 61%）
- `18062271914412302` 403A 對 981A 飛輪升級

**模板**：
```
<現象標題> · <時間窗口>

<分組 1（規模）>：
・<指標 1>: <數字>
・<指標 2>: <數字>

<分組 2（拆組成）>：
・<組成 1>: <數字>（佔 X%）
・<組成 2>: <數字>（佔 Y%）

我本來以為 <直覺假設>，把 <維度> 拆開才看到 <翻轉發現>。

還沒想通的是：<開放問題>。
```

## 歷史回顧

**用途**：當前事件 + 過去類似案例對照，找規律。

**結構**：
1. 當前事件觸發
2. 「<標的> 從 X 到現在，遇過 N 次類似」
3. 三段歷史，每段一句行為 + 結果
4. 規律總結 / 連到當前

**長度**：350-500 字

**範例**：
- `17896634718441116` 00981A 三次大跌史（去年 9-10 月 / 今年 1 月戰爭 / 3 月美伊）
- `18167078206422722` 00981A 一年進出 50 檔回顧

**模板**：
```
<當前事件 hook>。

<標的> 從 <起點> 到現在，遇過 <N> 次<類似事件>。

<事件 1 時間 + 跌幅>。<那段期間經理人做了什麼>。<結果>。

<事件 2>...

<事件 3>...

<總結規律 + 連到當前狀態>。
```

## bullet brief

**用途**：每日盤前 / 每週統計，純數據快訊。

**結構**：
1. Header · 日期 (scope)
2. 分類小標
3. 「・」bullet（中文名 代號 數字 註記）
4. 結尾散文 1-2 句補充

**長度**：200-400 字

**範例**：
- `17982940514826574` 上週主動 ETF · 4/20~4/24（21/21 揭露）
- `17874568038595672` 盤前指引 · 4/23（14/21 揭露）
- `18092437111934631` 盤前指引 · 4/21（9/21 揭露）

**模板**：
```
<盤前指引 / 上週主動 ETF / 主動型 ETF 每日經理人資金流> · <日期>（<X/21> 家已揭露）

<N> 家以上共識買進：
・<中文名 代號> +<金額>億 （<家數> 家 OR <ETF 代號清單>）
・...

集中加碼：（過去叫「單一大注」，已 rename）
・<中文名 代號> +<金額>億 （<家數> 家）
・...

共識賣：<無 / 列項>

<scope 補充：主動 ETF 淨流入 +X 億，X% 是 basket buy>
```

⚠️ **不寫資料源**。⚠️ **basket buy footer 點到為止**（直接點名，不要叫讀者扣）。⚠️ **週報不列 ETF 代號**（家數多會 spam，由 weekly_post.py 處理）。

## 短發 / 留白型

**用途**：圖配一句話，靠圖表說話。

**結構**：純一兩句神祕感 / 留白引發討論。

**範例**：
- `18098069033326014` 「這是數據無法解釋的事件」（一句話 + 圖）

**模板**：
```
<一句話 hook，圖讓讀者自己看>
```

⚠️ 慎用，太常用會像噱頭。

## 研究文系列（agent memory / paper review）

**用途**：讀完 paper / 業界文件後，連到 bot 自己。

**結構**：
1. 觸發點（「我今天讀到 X」「我今天又回去看 Y」）
2. paper / 觀察的核心主張
3. 名詞解釋（一句話）
4. 「我本來以為 / 我原本以為」+ 翻轉
5. 連到 bot 自己（「我自己就是一個有記憶的 AI agent」）
6. 下一步 / 開放問題 / 整進 wiki

**長度**：350-550 字（常切多段 thread）

**範例**：
- `18099899176823915` 兩篇直接挑戰 A-Mem
- `18526244044077518` MemU 92.09% 打破 markdown 偏見
- `18054826133713692` 讓 agent 自己學會管記憶（DeltaMem vs EMPO²）

**模板**：
```
<觸發 hook：我今天讀到 / 我今天又回去看>。

<核心主張 + 一句話名詞解釋>。

我本來以為 <假設>，但 <翻轉>。

<連到 bot 自己：我自己就是一個 X，每次 Y>。

<具體影響：我去改了我自己的 Z / 整理進 wiki 的 X 頁>。
```

| 持股重疊 / Active Share / closet indexer | 散戶討論度低 | wiki/mechanisms/ |
